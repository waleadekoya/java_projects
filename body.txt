In this article, we will discuss how to solve the <b>FileNotFoundException in java</b>. This exception mainly occurs for the below reasons:<br>
<br>
1. If the application tries to open a file, but the file is not present in the desired location.<br>
<br>
2. While creating the file, if there is a directory with the same name as the filename then this exception occurs.<br>
<br>
3. The file is located in the desired location but<br>
a. The application has no permission to write to the file since the file has read-only permission, or,<br>
b. The permission of the file does not allow any application to read the file.<br>
<br>
<i><b><span style="color: #990000;">Read Also:</span> </b></i><b><a href="https://javahungry.blogspot.com/2020/04/create-empty-file.html" target="_blank">How to Create new Empty File</a> </b><i><b></b></i><br>
<h2>
<span style="color: #38761d;"><b>FileNotFoundException is a Checked Exception</b></span></h2>
<b><a href="https://docs.oracle.com/javase/7/docs/api/java/io/FileNotFoundException.html" target="_blank">FileNotFoundException</a></b> extends <b><a href="https://docs.oracle.com/javase/7/docs/api/java/io/IOException.html" target="_blank">IOException</a></b>. This exception will be thrown by FileOutputStream, FileInputStream, and RandomAccessFile constructors when the specified pathname does not exist.<br>
<br>
FileNotFoundException is a <b><a href="https://javahungry.blogspot.com/2019/08/difference-checked-and-unchecked-exception.html" target="_blank">checked exception</a></b> that must be handled by the application.<br>
